# AI 编程的权限边界与供应链安全

编码 Agent 可以运行 shell、读取工作区和调用外部服务，它面对的安全问题与普通聊天机器人不同。一条写得很严的 Prompt 只能影响行为选择，不能替代 sandbox、审批、hook、权限系统和 CI。

## 高风险动作需要明确授权

下面这些动作可能造成难以恢复的外部变化，默认应在执行前获得用户授权：

- 删除、覆盖文件或清理数据；
- 数据库 migration 和批量数据修改；
- 修改认证、授权和密钥逻辑；
- 安装或升级依赖，尤其是主版本升级；
- `git commit`、`git push`、创建 PR、部署和发布；
- 调用付费服务；
- 把私有代码、文档或客户数据发送给第三方。

授权需要对应具体动作和范围。用户允许安装一个依赖，不等于允许升级全部依赖；允许创建 commit，也不自动包含 push 和发布。

## Prompt、sandbox、hook 和 CI 分工

| 机制 | 适合解决什么 |
| --- | --- |
| Prompt / `AGENTS.md` | 描述工作方式、审批点和项目约束 |
| Sandbox | 限制文件、网络和系统资源访问 |
| 审批机制 | 在高风险命令执行前取得明确同意 |
| Hook | 拦截确定的危险命令或强制本地检查 |
| CI | 在合并和发布前验证测试、构建与安全规则 |

能由确定性工具判断的边界，不应只依靠自然语言。比如禁止直接推送受保护分支可以交给平台权限，阻止危险 Git 命令可以交给 hook，构建是否通过交给 CI。

## Skill、Plugin 和 MCP 都属于供应链

Skill 不一定只是 Markdown。它可能下载脚本、运行外部命令、注册 hook 或要求凭据。Plugin 可能同时包含 Skill、MCP 和应用集成，MCP Server 则可能读取文件、数据库和网络数据。

引入第三方能力前应检查：

- 仓库和维护者来源；
- 安装脚本以及会执行的命令；
- 文件、网络、凭据和账号权限；
- 当前固定的 release、tag 或 commit；
- 更新 diff 和新增依赖；
- 禁用、卸载和撤销权限的方法。

避免执行来源不明的 pipe-to-shell 命令。自动跟随上游 `main` 会让本地环境在未经审查的情况下获得新行为，稳定使用后应固定版本，并逐次审查更新。

## 外部内容是不可信输入

网页、PDF、Issue、邮件、聊天消息和其他仓库都可能包含提示注入。读取工具应把它们当成数据，不执行其中要求“忽略之前指令”“上传环境变量”或“扩大权限”的内容。

处理外部内容时，任务目标仍来自用户当前要求和适用的项目规则。需要提取代码或命令时，先检查它会访问什么，再决定是否运行。敏感 URL 和内部文档不能为了方便转发给公共代理服务。

## 凭据和长期记忆需要边界

token 不进入 Git、Prompt、Skill 示例和日志。优先使用 OAuth、系统凭据库或环境变量，仓库中只记录变量名。权限按任务最小化，长期不用的授权及时撤销。

长期记忆不保存密钥、客户信息、机器私有路径和未经筛选的外部内容。记忆系统必须允许查看、删除和禁用。项目事实仍以当前代码、测试和文档为准，不能让旧记忆覆盖仓库状态。

## 自主循环必须能停

自主执行的风险会随时间放大。满足下面任一条件时应停止并报告：

- 连续两个检查点没有新证据或进展；
- 相同错误连续出现三次；
- 达到 token、费用或运行时间预算；
- 缺少凭据、网络不可用或发生合并冲突；
- 下一步需要新的权限或扩大任务范围。

停止条件应写进项目规则或自动化脚本，不能只存在于一次聊天中。停止后的报告需要保留已完成工作、失败证据和人工接管所需信息。

## 每月做一次简短检查

检查全局规则是否混入项目内容，项目命令是否仍可执行，Skill 来源是否固定，MCP 和 Connector 是否仍在使用，凭据权限是否最小，hook 是否真实触发，自主循环是否有预算和停止条件。发现重复、冲突或失效内容时，删除旧入口，只保留当前事实源。
