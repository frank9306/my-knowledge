# QuickNav：我的个人导航站浏览器扩展

作为一个经常需要访问各种网站和工具的开发者，我发现浏览器自带的书签功能并不能很好地满足我的需求：书签层级太深，查找不方便，界面也不够直观。于是我决定开发一个浏览器扩展，将新标签页改造成一个现代化的个人导航站。

这就是 **QuickNav** 的诞生背景。

**项目地址**: [https://github.com/frank9306/quick-nav-extension](https://github.com/frank9306/quick-nav-extension)

![QuickNav：我的个人导航站浏览器扩展 image 1](/images/imported/quicknav-browser-extension/image-01.webp)

## 主要功能

### 新标签页导航

每次打开新标签页时，自动展示我的个人导航站：

*   左侧是分类列表，可以按类别筛选网站
*   右侧是卡片式的网站展示，每个卡片显示网站的标题、描述和标签
*   顶部有搜索框，支持搜索标题、描述和标签
*   界面采用现代化设计，简洁美观

### 快速添加网站

提供了两种便捷的添加方式：

**右键菜单添加**

在任何网站上右键，选择"添加到导航站"，就可以快速将当前网站收藏到导航站。更贴心的是，右键点击链接也可以直接添加那个链接，不需要先打开。

**弹窗快速添加**

点击浏览器工具栏的扩展图标，会弹出一个小窗口，自动获取当前页面的信息，填写后即可添加。

### 智能辅助

扩展内置了一些智能功能，让添加网站更方便：

*   自动识别网站的标题和描述
*   根据网站的 URL 和内容智能推荐合适的分类
*   自动生成相关的标签建议

### 数据管理

在扩展的设置页面（Options），可以进行完整的数据管理：

*   批量添加、编辑、删除导航项
*   导出数据为 JSON 文件，方便备份
*   导入之前导出的数据，轻松迁移
*   一键重置恢复默认数据

所有数据都存储在本地，使用 Chrome Storage API 实现跨页面同步，确保在新标签页、弹窗和设置页面看到的都是最新数据。

## 为什么选择 Plasmo 框架

开发这个扩展时，我选择了 [Plasmo](https://www.plasmo.com/) 框架，这是一个现代化的浏览器扩展开发框架，它让扩展开发变得像开发普通网页应用一样简单。

### Plasmo 的优势

**1\. 极简的项目结构**

传统的浏览器扩展开发需要手动编写复杂的 `manifest.json` 配置文件，配置各种页面入口、权限、脚本等。而 Plasmo 通过约定优于配置的理念，让你只需要关注功能开发：

*   创建 `newtab.tsx` 文件，自动成为新标签页
*   创建 `popup.tsx` 文件，自动成为扩展弹窗
*   创建 `options.tsx` 文件，自动成为设置页面
*   创建 `background.ts` 文件，自动成为后台脚本

不需要手动配置，Plasmo 会自动生成正确的 manifest 文件。

**2\. 现代化的开发体验**

Plasmo 内置了我们熟悉的现代前端工具链：

*   **React + TypeScript**：可以用 React 和 TypeScript 开发扩展，就像开发普通的 React 应用
*   **热重载**：修改代码后自动刷新扩展，无需手动重新加载
*   **自动构建**：一条命令就能构建生产版本，自动优化和压缩

**3\. 开箱即用的功能**

Plasmo 提供了很多实用的内置功能：

*   **Storage API 封装**：简化了 Chrome Storage 的使用
*   **消息传递工具**：用于页面间通信
*   **类型定义完整**：完善的 TypeScript 类型支持

**4\. 跨浏览器支持**

一套代码可以构建出适配多个浏览器的扩展包：

*   Chrome / Edge（基于 Chromium）
*   Firefox
*   Safari

只需要运行不同的构建命令，Plasmo 会自动处理不同浏览器的差异。

**5\. 活跃的社区**

Plasmo 有完善的官方文档和活跃的社区，遇到问题很容易找到解决方案。

### 使用 Plasmo 的开发流程

开发 QuickNav 的过程非常流畅：

1.  **初始化项目**：使用 Plasmo CLI 快速创建项目
2.  **开发功能**：用 React 开发各个页面，像写普通网页一样
3.  **实时预览**：运行 `pnpm dev`，代码改动实时生效
4.  **构建发布**：运行 `pnpm build`，生成可发布的扩展包

整个过程没有繁琐的配置，可以专注于功能实现。

### 我的感受

使用 Plasmo 开发扩展的体验和开发现代化的 Web 应用几乎一样，大大降低了浏览器扩展开发的门槛。如果你熟悉 React，那么用 Plasmo 开发扩展几乎没有学习成本。

相比传统的扩展开发方式，Plasmo 让我节省了大量配置和工具链搭建的时间，能够更快地实现想法。

## 技术栈

QuickNav 的技术选型：

*   **框架**：Plasmo Framework 0.90.5
*   **前端**：React 18 + TypeScript
*   **样式**：纯 CSS，现代化设计
*   **存储**：Chrome Storage API（支持跨页面同步）
*   **包管理**：pnpm
