# React Flow (子流程)

`parentNode`  属性已弃用！我们在版本 11.11.0 中将 `parentNode` 选项重命名为 `parentId` 。旧属性仍然受支持，但将在版本 12 中移除。

子流程是一个节点内的流程。它可以是独立的流程，也可以是与父节点外其他节点相连的流程。此功能也可用于分组节点。

节点顺序 在 nodes / defaultNodes 数组中，父节点必须出现在其子节点之前，才能正确处理。

我们通过传递样式选项来设置父节点的固定宽度和高度。此外，我们将子节点范围设置为 'parent' ，这样我们就不能将子节点移出父节点。

### **使用特定于子节点的选项**

*   **当你移动父节点时，你会发现子节点也会随之移动。使用 parentId 选项将一个节点添加到另一个节点上，它只会做一件事：相对于其父节点定位它。**
*   **从标记的角度来看，子节点并不是真正的子节点。你可以将子节点拖动或定位到其父节点之外（当未设置 extent: 'parent' 选项时），但当移动父节点时，子节点会随之移动。**
